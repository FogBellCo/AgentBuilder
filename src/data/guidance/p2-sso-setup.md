# Working with Internal Data (P2)

## What this means

Your data requires UCSD Single Sign-On (SSO) authentication. Only authenticated UCSD users can access and process this data with AI tools.

## Setting up access

1. **Verify your UCSD SSO** — Log in at [sso.ucsd.edu](https://sso.ucsd.edu) to confirm your credentials work
2. **Check system access** — Ensure you have permissions for the specific system where the data lives
3. **Use authenticated AI channels** — When using AI tools, make sure you're connected through UCSD's authenticated environment

## Important considerations

- AI outputs derived from P2 data should only be shared with other authenticated UCSD users
- If you need to share results externally, consider whether the output needs to be de-identified
- Always use UCSD-approved AI platforms for processing internal data

## Need help with SSO?

- [UCSD IT Help Desk](https://support.ucsd.edu) — For SSO login issues
- [Active Directory Services](https://blink.ucsd.edu/technology/security/) — For permission requests
